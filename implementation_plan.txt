# Project Sentinel - Phase 1 MVP Implementation Plan

The objective of this Phase 1 MVP is to build a lightweight machine learning text classifier capable of analyzing comments and assigning a toxicity score and label.

## Proposed Changes

### c:\Projects\Sentinel – AI-Powered Toxicity Detection & Moderation System\

#### [NEW] [requirements.txt](file:///c:/Projects/Sentinel%20%E2%80%93%20AI-Powered%20Toxicity%20Detection%20&%20Moderation%20System/requirements.txt)
Will define the necessary Python libraries for this phase.
* `scikit-learn`: For the TF-IDF vectorizer and Logistic Regression models.
* `pandas`: For dataset handling.
* `datasets`: To seamlessly download the Jigsaw Toxic Comment dataset from Hugging Face.
* `joblib`: For efficient serialization of our model components.

#### [NEW] [train.py](file:///c:/Projects/Sentinel%20%E2%80%93%20AI-Powered%20Toxicity%20Detection%20&%20Moderation%20System/train.py)
This script will act as the training pipeline:
1. Load a scaled-down subset (e.g., 10,000 samples) of the Hugging Face `jigsaw_toxicity_pred` dataset to enable fast training.
2. Preprocess the text data (basic cleanups).
3. Build a pipeline with `TfidfVectorizer` and `LogisticRegression`.
4. Train the model using the binary "toxic" label.
5. Save the trained pipeline to `toxicity_model.pkl`.

#### [NEW] [predict.py](file:///c:/Projects/Sentinel%20%E2%80%93%20AI-Powered%20Toxicity%20Detection%20&%20Moderation%20System/predict.py)
A testing script intended for inference and ensuring the output matches the required format:
1. Load the saved `toxicity_model.pkl`.
2. Accept a text input (e.g., via command-line arguments).
3. Determine the prediction class (`toxic` vs `non-toxic`) and extract the actual toxicity probability score.
4. Output the result in the exact JSON format specified in the design (`{"toxicity": ..., "label": ... }`).
